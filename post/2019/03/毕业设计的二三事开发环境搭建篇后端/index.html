毕业设计的二三事——开发环境搭建篇（后端）

2019年3月23日
| 毕业设计

不是教程~， 单纯的对后端开发环境的选用以及配置过程的问题进行总结

关于技术选型

选用lavarel（php）框架。

  • 但是基于以前有过php以及CI（php）框架的使用经验，上手较快
  • 相较于CI，lavarel的使用更加优雅，且受欢迎度以比较高，对于大中型项目而言，lavarel更能胜任

关于本地开发环境的配置

  • window10
  • virtual—box
  • vagrant
  • lavarel/homestead (lavarel的集成开发盒子)

注意事项（后续补充）

  1. vagrant安装lavarel/homestead，会产生文件夹.vagrant.d,默认路径是c盘，自定义请执行以下命令：
    • setx VAGRANT_HOME “E:\VirtualBox_HomesteadBoxs”
    • setx VAGRANT_HOME “E:\VirtualBox_HomesteadBoxs” /M
  2. homestead文件的内容需要与.ssh,.vagrant,.vagrant.d同级
  3. vagrant box list 命令后，出现可用的box列表，如果是通过单独安装的box，可能会出现 盒子名后面的版本号 是0，解决方案：
    • .vagrant.d\boxes\laravel-VAGRANTSLASH-homestead\ 这个目录下的 0改成指定版本数字就可以了，ex：7.1.0

  4. .vagrant\machines\这个路径下的文件夹名决定了生成的虚拟机的名字，如：E:\VagrantBoxs_Homestead\.vagrant\machines\homestead-7, 虚拟机名：E:\virtualBox VMS\homestead-7。

  5. vagrant up项目卡在 private key 可能出现ssh登录错误的问题，针对不同的原因，诸如ssh本身错误，亦或是配置文件错误等等，网上的解决方案很多，我能提醒的是，务必请注意：虚拟机的ssh默认端口:2222，可能存在被占用的情况，可以在VagrantFile的文件中对ssh设置，该文件具体配置项请参照vagrant官方文档，也可以对homestead.yaml中的port选项进行设置，两者都可以额外开通信端口

    # 这里我的解决方案是对ssh重设
    # vagrantfile
    config.vm.network "forwarded_port",
        guest: 22,
        host: 3333,
        id: "ssh"
