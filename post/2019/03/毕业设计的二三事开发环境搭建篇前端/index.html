毕业设计的二三事——开发环境搭建篇（前端）

2019年3月23日
| 毕业设计

本篇主要说明前端开发环境的选择&&搭建过程中遇到的问题&&相应的解决思路，我并不会给出具体的解决方案，我会给出思路，因为方案其实不是很重要，解决问题的方向才是最重要的，这个过程中我踩了很多的坑~

技术栈的选择

说一下我这里选择的前端的技术栈: Webpack + Typescript + React + Mobx + AntDesign 选择的原因么，两点：

  • webpack是为了快速开发，丰富的插件提供了各式各样的支持。
  • Typescript 项目的开发更加的严谨，减少不必要的错误。
  • react的拓展性能相较于vue更好些，作为大型的项目来开发选择react准没错，当然，vue3也是用ts重写了的，但是由于其提供的全家桶开发更倾向于与单页应用，所以我拒绝。- - - mobx，我不会redux，而mobx在实习期间用过，而且使用更简单。
  • AntD, 现成的轮子不用还自己造，嘿嘿，更何况这套组件的设计本身就非常棒可以看看源码去学习一下。另外现造轮子其实没多大意义, 除非兴趣所在。
  • 我以前在实习的时候，曾经在公司中使用过这种技术栈，所以尝试自己搭个架子，现在看来还不错

错误以及解决思路

webpack4

这里并不讨论webpack4的性能变化，只是如果在配置过程中出现了错误，可以尝试从以下方向着手。

  1. webpack本身功能的变化：随着webpack升级到第四版本，webpack自身的变化最大的是：

    • 很多功能所在的位置并不与webpack3相同，最明显的区别就是出现了新的optimization的配置项，并将原本的splitChunks（webpack3中并不叫这个名字）的功能移到这个配置项中.这点极其重要。
    • webpack4.resolve有很多配置项需要着重关注，alias,extensions之类的，大概会很常用
    • 并且一些3版本的插件功能可能已经集成到了webpack4中。

  2. 对应的插件/loader在第四版本的变化：在配置过程中我遇到的最尴尬，莫过于插件在webpack4的兼容性的问题。

    • 同名不同版本的插件在配置上的分歧： "clean-webpack-plugin": "^2.0.1" 这个插件 ，其1.x和2.x的版本配置要求是完全不一样的。
    • 插件版本能否正确运行在当前版本的webpack： 貌似是这个，"typings-for-css-modules-loader": "^1.7.0"，真心坑了我很久，最后在StackOverflow上找到的答案，其最新版本对webpack4的支持竟然反不如1.x版本的，难受呀。

awesome-typescript-loader && babel && antd

这个其实没什么好说的，照着官网配就可以了，然而需要说明的是，在配置项目的时候，很多情况离不开 诸如babel-xxx或者@babel/xxx之类的插件 然而我们要注意的是，这是一个ts项目，用的是 awesome-typescript-loader来编译成js，那就意味着，babel插件怎么办？，别急，这个插件支持babel配置，很方便吧！ 在options的选项中：

    useBabel: true,
    babelCore: "@babel/core",
    babelOptions: {
        // 不需要babelrc文件
        babelrc: false,
        presets: [],
        plugins: []
    }

然而我当时并没有意识到，结果在下面又加了个babel-loader配置，然而并没有用。所以请务必注意插件本身的拓展功能是否是项目中需要的！

  1. 这个插件本身集成了很多功能，诸如 TsConfigPathsPlugints的文件路径功能，类似 webpack.resolve.alias，可以配置快捷名称如@component来指向那些又臭又长”../../../component/Page”之类的路径名,所以在安装插件时优先考虑这种功能是否已经被集成到某个插件中的情况，避免无用功。
  2. 这点上面提到过，支持babel的设置，诸如babelCore的是设置（如使用babel7），或者presets、plugins等功能
    • 这里提一下antd的按需引入问题，原本我使用ts-import-webpack-plugin(好像是这个名…)这个插件的，结果忙了两三天，完全无用，撞完南墙得我决定回头(这里说一下原因：我原本不知道awsome-typescript-loader居然支持babel，而官网推荐的是babel-plugin-import很明显要用到babel，于是我就很纠结了)······，安装babel-plugin-import，按照官网配置就可以了
  3. 最后一个算是印象很深刻的问题 我在使用 import() ,webpack在运行时，说是无法识别这个方法，这就很尴尬了。这个时候，我们需要一个能够让webpack4识别 import() 这种写法的插件,铛铛~，babel7的@babel/plugin-syntax-dynamic-import插件，说一下，这个插件是对应babel版本的，务必请注意哦！

tsconfig.json的配置

这其实没什么说的，照着官网或者某某的博文，配置一下就ok了，另外注意，个别插件需特殊配置，ex: awsome-typescript-loader

尾语

可能还有其他的要点，暂时没想起来，哈哈~，对了，配置过程中的语法错误是最低级，也是最容易忽略的，请务必注意！！！
